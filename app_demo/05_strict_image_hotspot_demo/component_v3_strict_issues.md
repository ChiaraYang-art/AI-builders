# component_v3_strict 开发记录问题清单

这版严格按 Figma 已存在 frame 和 prototype 线实现，没有再补未设计页面。开发中遇到以下需要后续确认的问题：

1. `首页小芽说话框` 在 Figma 里只连到 `出门邀请-lightwalk`，但按你的说明已改成随机进入四种邀请页。【没问题，后续都这么做】
2. 所有底部导航的“散步”栏按你的说明改成随机进入四种邀请页，即使 Figma 里不是每个“散步”图标都连了线。【没问题，后续都这么做】
3. `出门邀请-lightwalk` 和 `出门邀请-soundwalk` 里的“今天算了吧”没有读取到 prototype 线，所以严格版里暂时不做跳转。【现在连上了】
4. `散步完成反馈` 只读取到“查看散步日记”按钮连到流水账；“返回首页”没有读取到线，所以严格版里没有给它加跳转。【现在连上了】
5. `分享散步卡片` 只读取到返回按钮连回普通小作文页；保存图片和分享按钮没有读取到线，所以严格版里暂时不做动作。【是的，目前这两个界面我们还没做，按理说保存图片会出现一个弹窗提示保存成功，分享按钮会出现一个底部弹窗，用户可以选择分享到什么APP。目前我们没精力做，如果你能给我们生成一下也可以，不生成也没关系，这版先这样。】
6. `小芽日记-小作文-colorwalk` 没有读取到分享按钮线，严格版只保留返回和 tab 切换。【这个其实是有的，但是这个页面很长，分享按钮在frame的clip content外面，所以你可能看不到。正常交互了流程下，用户需要下滑，看到分享按钮之后就可以点击了。】
7. `附近` 页面目前只按底部导航处理，没有读取到页面主体卡片的出门/发现类交互线。【是的，这一步我们没精力设计了】
8. 因为目标是严格保留设计稿，这版使用 Figma frame 图片作为视觉层、透明热区作为交互层；如果要让每个卡片和文字都可被代码同学直接改，需要再做一轮“逐页面组件化还原”，工作量会更大。
