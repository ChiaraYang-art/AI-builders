### 2.1 先回答：后端现在能「多段录音 + 总结」吗？

| 能力 | 是否已有 | 说明（设计理解用） |
|------|----------|-------------------|
| **同一次散步存多段录音** | ✅ 已有 | 在 Sound Walk 里可多次点「录一段环境音」，每段都会 `POST /walk/audio`，后端存在该次 `walk_id` 的 `audios[]` 里 |
| **每段录音的 AI 标签 + 一句总结** | ✅ 已有（轻量） | 每上传一段返回 `sounds`（如 birdsong、traffic）和中文 `summary`，并写入本次散步的「事件时间线」 |
| **散步结束生成总日记** | ✅ 已有 | 「完成散步」→ `POST /walk/diary`，把**所有照片 + 所有录音 + 路上事件**交给大模型，生成 `title` / `timeline`（流水账）/ `essay`（小作文） |
| **地图上的「轨迹总结」** | ⚠️ 仅文案 | 有 `map_summary` 一句（如「1.8km · 2 张照片 · 3 段声音」），**不是真实 GPS 路线**；地图页仍是 Figma 静态 SVG |
| **Sound 页列出每一段** | ❌ 未做全 | App 只显示「已记录 N 段环境音」，**没有** PRD 10.5 里的「每次录音时间 / 波形 / 声音贴纸列表」 |
| **听音频文件做识别** | ⚠️ Demo 简化 | 后端保存了真实音频文件，但识别主要靠 LLM 按场景猜标签（未接专用音频模型），Demo 够用、非成片级 |

**和硬件麦克风的区别**：设备上的麦克风仍只用于判断「安静 / 较响」和室内外推断；**散步录音是手机浏览器录的**，两套能力。

## 4. 今天下午验收结论（设计相关）

### 4.1 已通过 ✅

- **首页 / 散步页实时更新**：时间戳、lux、温湿度、运动状态会跳变（说明联调 OK）。
- **小芽 AI 说话**：中文诗意长句（非英文兜底）。
- **四种散步入口**：邀请 → 散步 → 暂停 → 完成 → 日记 / 流水账 / 小作文 / 地图 / 图鉴 页面可点通。
- **Color Walk**：拍照后网格有预览、进度增加。
- **Sound Walk**：浏览器录音可上传。
- **图鉴**：完成散步后对应项可显示「已解锁」。

### 4.2 需产品/运维知晓 ⚠️

- **语音 TTS（CosyVoice）**：下午联调中曾出现百炼 **免费额度用尽**（`AllocationQuota.FreeTierOnly`），表现为 `tts=error`、硬件不播 MP3。  
  - 与 UI 无关，恢复额度或开通按量付费后即可。  
  - 文案、传感器、页面刷新不受影响。

### 4.3 建议设计同学自测的 5 分钟路径

1. 打开首页，看 `最近更新` 是否在变。  
2. 点说话框 → 进邀请页 →「出门！」→ 散步页。  
3. Color：拍 1–2 张（偏绿更易识别）→ 看照片格。  
4. Sound：录几秒环境音。  
5. 暂停页「完成散步」→ 日记 / 图鉴 / 地图各点一次。

## 5. 页面上看到的中文标签 ↔ 数据来源（方便对齐文案）

设计在散步页看到的归纳标签，来自后端字段，不是 GPS：

| App 展示（示例） | 数据来源 | 含义（简化） |
|------------------|----------|----------------|
| 在室外 / 在室内 / 未知 | `place` | 由光照 + 是否走动 + 声音环境 **推断**，非定位 |
| 光照很弱 / 一般 / 明亮 | `lux` 数值分段 | BH1750 光照传感器 |
| 安静 / 较响 / 未知 | `sound_state` | 麦克风环境音统计 |
| 温度 xx°C 湿度 xx% | ENV-Pro | 温湿度 |
| 小芽气泡长文 | `speech_full` | AI 根据上述状态生成，每次上报可能变 |

**走查建议**：长文案请按「最少 2 行、最多约 5 行」留弹性；传感器一行建议不换行溢出（当前为短标签拼接）。

## 8. 与 Figma 对齐时建议优先级

| 优先级 | 页面 | 原因 |
|--------|------|------|
| P0 | 首页、四种邀请页、四种散步页、暂停/完成 | Demo 主路径 + 实时数据最多 |
